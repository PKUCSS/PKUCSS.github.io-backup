SRAM,DRAM and Related Concepts

SRAM

Static Random Access Memory,静态随机访问存储器，属于易失性存储器，需要保持通电。访问速度快、密度较低，成本较高，一般用作Cache。

DRAM and Related Concepts

Naive DRAM

Dynamic Ramdom Access Memory,动态访问存储器，采用单管+电容器存储每个位，需要定期刷新。密度高、速度比SRAM慢、价格较低，常用作主存。

Fast Page Mode DRAM(快速页模式)

Naive DRAM(N行N列M位)，每读M为需要指定一次行地址和一次列地址，需要一个RAS/CAS周期，而FPM DRAM用NXM SRAM作为缓存保存行，在读一行到寄存器后，只需要CAS(列选择)来访问该行的其他M位存储块，在CAS变化时RAS保持有效，不需要频繁地指定RAS,读取速率较快。

EDO DRAM & Burst EDO DRAM

Extended Data Out DRAM(拓展数据输出内存)，是FPM DRAM的改进版本，在输出本列地址对应的数据时就开始下一列地址的选通，把不同列的访问时间重叠起来，速度比FPM DRAM更快。

BEDO RAM（Burst Extended Data Output RAM）：突发扩充数据输出随机存储器 ，a faster type of EDO, memory with an improved outcome. BEDO RAM gained its speed by using an address counter for next addresses and a pipeline stage that overlapped operations.BEDO RAM reads data from a package, which means that after receiving addresses each of these three pieces of information is read for one cycle of the timer, and the CPU reads the data packet in a 5-1-1-1.

SDRAM & DDR

SDRAM(Synchronous DRAM,同步动态访问随机存储器)，是带有同步接口的DRAM,通过一个相同的时钟和CPU锁在一起，使DRAM和CPU能够共享一个时钟周期，以相同的速度同步工作。基于于双存储体结构，内含两个交错的存储阵列，当CPU从一个存储阵列访问数据时，另一个就已为读写数据做好了准备，通过这两个存储阵列的紧密切换（流水线操作），读取效率就能得到成倍的提高。

Double Data Rate双倍速率,DDR SDRAM=双倍速率同步动态随机存储器，人们习惯称为DDR，在SDRM的基础上改进而来，在时钟的上升沿和下降沿都能进行数据传输，速率是系统时钟周期的两倍。

Main Memory Packaging

自高到低：Channel → DIMM → Rank → Chip → Bank → RowxColumn,Row Buffer

每个channel 都需要配有一组内存控制器。而每个 channel 中能够有很多组 DIMM(Dual In
line Memory Module) DIMM 就是内存条，当然也有 SIMM 的（现在很少见） 。 DIMM 上每个面
上的一系列 chips 组成了一个 rank 。rank指的是连接到同一个 cs(Chip Select ，片选 的所有内存颗粒 chips ，内存控制器能够对同一个 rank 的所有 chips 同时进行读写操作，而在同一个 rank 的 chip 也分享同样的控
制信号。每一个chips 又可以分为多个 banks ，这些 ban k 可以交替访问。每个 bank 都是由内存单元组成的矩阵，矩阵有行地址和列地址。Row Buffer 用于缓存同一行地址取出的连续内存单元。

Cache and ESI protocol

Cache

Cache不多赘述，记住3C Analysis(三种miss, Compulsory ,Conflict and Capacity),Write back和Write through(写返回和写穿透，即写命中时不写回/写回下一级)，Write Allocate和No Write Allocate(写分配和不些分配，写失效时更新/不更新Cache)。

Snoopy Cache

多核需要保持缓存的一致性，有两种方法：

  • Snooping Solution:基于总线窥探的缓存一致性协议，每个对数据的读/写需要都从总线广播到各个处理器，如果处理器对应缓存中有相应缓存块，则根据协议做相应的状态转变。常用。
  • Directory-Based:基于目录的缓存一致性协议，一个中央目录控制的角色。原来向总线发送的的消息改为向目录持有者发送消息。而目录持有者负责改变每个处理器上缓存数据的状态。这样可以减少总线的使用。适用于核数特别多的大型机，不多见。

Snoopying For DMA

  • DMA Read Memory → Disk
    • Cache State: Address not cached → No action
    • Cache State: Address cached,unmodified → No action
    • Cache State: Address cached,unmodified → Cache intervenes(写回，再传给Disk)
  • DMA Write Disk → Memory
    • Cache State: Address not cached → No action
    • Cache State: Address cached,unmodified → 将对应Cache标记为invalid或修改写进新值
    • Cache State: Address cached,modified → 将缓存块写回主存，缓存块标记为invalid，再进行DMA写主存

多核缓存ESI协议

每个Cache Block处于三种状态之一：

  • Exclusive(或称modifeid)，已修改）：块已在缓存中已被修改，缓存中的数据与后备存储器（例如内存）中的数据不一致。具有“M”状态的块的缓存在该块被替换时需要将其中的内容写回后备存储
  • Shared，块未被修改，并在至少一个处理器的缓存中以只读状态存在。高速缓存可以将其替换而不将其中的数据写回后备存储
  • Invalid:该块不存在于当前缓存中，或者因为总线请求而被标记为无效。如果要将某个块存储在该缓存中，则必须首先从内存或另一个高速缓存中获取该块

这些一致性状态通过高速缓存和后备存储之间的通信进行维护。 当缓存中的某个块被读或写时，或者当缓存通过总线接收到其他缓存发出的读写信号时，它需要据此来做出动作并调整自己的状态。

如何将TLB访问和Cache访问重叠起来

对于使用虚拟地址的处理器发出存储请求时，需要将虚拟地址转换为物理地址。在转换过程中， TLB 的工作是将虚拟页号转换为页帧号， 但是转换中， 地址中的页内偏移不会发生改变。如果访问 cache 的索引只需要使用地址中的页内地址，则可以不需要经过地址转换直接访问，即实现 cache 的访问和 TLB 访问同时进行。为了使得 cache 的索引只使用页内地址，可以增大页面大小或者提高 cache 的相联度。
